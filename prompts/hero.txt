## Movimiento principal de productos

Quiero que el producto o productos principales se muevan desde la derecha hacia el centro durante el scroll.

La animación debe ser aproximadamente así:

Estado inicial:

* producto ubicado visualmente hacia la derecha
* producto ligeramente desplazado en X positivo
* puede tener una escala ligeramente menor
* opacidad completa o casi completa
* sombra acompañando al producto

Durante el scroll:

* producto se desplaza progresivamente hacia el centro
* el movimiento debe sentirse natural, no mecánico
* puede tener una ligera variación en Y para sensación flotante
* la sombra debe acompañar con un movimiento más sutil
* si hay dos productos, deben tener velocidades o offsets ligeramente distintos para crear profundidad

Estado final:

* producto centrado o casi centrado
* producto con protagonismo claro
* composición estable
* elementos finales rescatados del antiguo ScrollTrigger visibles o integrados debajo/alrededor según convenga

## Reglas técnicas para GSAP

Usa GSAP de forma limpia.

Si el proyecto está en React, usar:

* `useRef`
* `useEffect` o equivalente según stack
* `gsap.context()` para limitar scopes
* cleanup correcto con `ctx.revert()`
* `ScrollTrigger` registrado una sola vez donde corresponda

Ejemplo conceptual, adaptar al proyecto real:

```js
gsap.registerPlugin(ScrollTrigger);

const ctx = gsap.context(() => {
  gsap.timeline({
    scrollTrigger: {
      trigger: sectionRef.current,
      start: "top top",
      end: "bottom top",
      scrub: 1,
      pin: false,
      invalidateOnRefresh: true,
    },
  })
  .fromTo(productRef.current,
    {
      xPercent: 35,
      yPercent: 0,
      scale: 0.96,
    },
    {
      xPercent: 0,
      yPercent: -4,
      scale: 1,
      ease: "none",
    }
  );
}, sectionRef);

return () => ctx.revert();
```

No copies este código literalmente si no encaja. Úsalo como intención técnica.

## ScrollTrigger

Quiero una animación elegante, pero no una sección excesivamente compleja.

Evitar por ahora:

* pinned sections largas
* scroll-jacking agresivo
* timelines demasiado grandes
* animaciones dependientes de muchos cálculos manuales
* mezclar demasiadas responsabilidades en un solo componente

Aceptar:

* `scrub` suave
* movimiento horizontal controlado
* parallax sutil
* escala leve
* entrada progresiva de elementos finales
* sombras o texturas moviéndose con menor intensidad

Config recomendada inicial:

```js
scrollTrigger: {
  trigger: sectionRef.current,
  start: "top center",
  end: "bottom center",
  scrub: 1,
  invalidateOnRefresh: true,
}
```

Ajusta `start`, `end` y `scrub` según el layout real.

## Responsive

La animación debe funcionar bien en desktop, tablet y mobile.

En desktop:

* movimiento completo de derecha a centro
* parallax sutil permitido
* productos con presencia visual fuerte

En tablet:

* reducir desplazamiento horizontal
* mantener fluidez
* evitar que el producto tape textos importantes

En mobile:

* simplificar animación
* menor desplazamiento
* quizá solo `y`, `opacity` o escala sutil si el movimiento lateral rompe el layout
* evitar pinned sections
* respetar rendimiento

Usa `matchMedia` de GSAP si tiene sentido:

```js
ScrollTrigger.matchMedia({
  "(min-width: 1024px)": function () {
    // animación desktop
  },
  "(min-width: 768px) and (max-width: 1023px)": function () {
    // animación tablet
  },
  "(max-width: 767px)": function () {
    // animación mobile simplificada
  },
});
```

Adáptalo al stack real y a la versión actual de GSAP.

## prefers-reduced-motion

Debe respetarse `prefers-reduced-motion`.

Si el usuario tiene reducción de movimiento activada:

* no usar animaciones de scroll complejas
* dejar el producto en posición final o en una posición estable
* evitar transformaciones continuas

## Performance

Prioriza rendimiento.

Requisitos:

* no cargar assets innecesarios
* no reintroducir vídeo
* no usar blur pesado en scroll
* no animar propiedades que fuercen layout
* animar principalmente `transform` y `opacity`
* evitar `top`, `left`, `width`, `height` animados
* limpiar ScrollTriggers al desmontar
* evitar duplicados al hacer hot reload
* comprobar que no quedan listeners antiguos
